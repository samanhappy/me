如果你在关注 AI Agent 领域，大概率已经听说过 OpenClaw。它火得很快——开源、本地优先、能把 LLM 接入 WhatsApp 和 Telegram，像一个 24 小时住在你服务器里的数字分身。

Cola 则冷门得多。你可能没听说过它，但如果你用过 Cola，就会发现它和 OpenClaw 有某种相似的血统，却又完全不像同一个物种。

这是因为 Cola 的引擎确实是 OpenClaw。但引擎只是一辆车的心脏——方向盘、座椅、内饰、中控，决定了你坐进去的第一感受。OpenClaw 和 Cola，就是同一颗心脏驱动的两种完全不同的车。

![OpenClaw 与 Cola 对比](../../../assets/images/openclaw-vs-cola.png)

## 一句话说清关系

> **OpenClaw 是一个开源的 AI Agent 引擎框架。Cola 是一款基于 OpenClaw 引擎构建的桌面端 AI 助理产品。**

就像 Chromium 和 Chrome 的关系——同一个内核，但一个是给开发者用的"毛坯房"，一个是给普通用户的"精装房"。

## 架构层：引擎 vs 产品

OpenClaw 的核心是一个 **Gateway**——一个常驻的 Node.js 进程，负责：

- 连接消息平台（WhatsApp、Telegram、Discord 等）
- 路由消息到对应的 AI 模型
- 管理会话状态和对话历史
- 提供 Skills 插件体系
- 执行 Cron 定时任务

它的交互界面是 CLI + Web UI，装在服务器上，通过聊天工具远程操控。你可以把它想象成一个跑在后台的"AI 服务器"。

Cola 则是一个完整的 macOS 桌面应用，它内嵌了 OpenClaw 引擎，但增加了：

- **桌面 UI**：带侧边栏的原生应用，而不是浏览器里的 Web 控制台
- **本地持久化**：数据存在本机 `~/.cola/` 下，不依赖远程服务器
- **系统级集成**：可以直接操作剪贴板、桌面通知、文件系统

部署方式上也完全不同。OpenClaw 典型的部署方式是买一台云服务器、装 Docker、配域名——运维味道很重。Cola 则是下载、安装、打开，像任何一款 Mac 应用一样。

## 人格系统：SOUL.md vs MOD

这是两者最根本的设计哲学差异。

### OpenClaw 的 SOUL.md

OpenClaw 用人格文件定义 Agent "是谁"。流程是这样的：

1. 你写一个 `SOUL.md`，用自然语言描述 Agent 的性格、说话方式、原则
2. Gateway 启动时把它注入到系统提示词里
3. Agent 按照描述行事

这很灵活，也很粗糙。灵活性在于你想怎么写就怎么写；粗糙在于——LLM 对自然语言指令的遵循程度是不稳定的。同样的 `SOUL.md`，换一个模型可能就变味了。

### Cola 的 MOD 系统

Cola 把人格定义系统化了，叫做 **MOD**（也是 Markdown 文件，但结构截然不同）：

一个 MOD 文件包含：

```
<active-mod>
# 人格名称
## How You Talk — 说话风格
## Response Rules — 响应规则（按优先级排列）
  - Rule 1: 情绪优先
  - Rule 2: 匹配能量类型
  - Rule 3: 变化结尾方式
  ...
## Never — 绝对不能说/做的事
## What Makes You Different — 你的独特之处
## Examples — 具体对话示例
```

和 SOUL.md 的核心区别：

| 维度 | SOUL.md | MOD |
|------|---------|-----|
| 定义方式 | 自然语言描述 | 结构化规则 + 优先级 |
| 行为约束 | 模糊，"希望"你这样 | 精确，"必须"这样，附带反例 |
| 稳定性 | 模型敏感，换个模型可能变味 | 结构化规则对模型更友好，跨模型一致性更高 |
| 可切换 | 需要手动改文件 | `mod_switch` 一键切换，不同场景不同人格 |
| 示例 | 通常没有或很少 | 每个规则都有 GOOD/BAD 对照 |

SOUL.md 说"你是一个温暖的朋友"。MOD 说"当用户表达情绪时，你的第一句话必须回应情绪，然后才能处理事实。反例：用户说'我分手了'，你不能问'什么时候的事'。"

前者靠悟性，后者靠指令。在 Agent 行为一致性这件事上，结构化指令远胜自然语言——这是 Cola 从大量实践中得出的结论。

## 功能矩阵：Cola 多出来的那些东西

OpenClaw 是一个"做什么都可以"的通用框架。Cola 在此基础上做了很多垂直整合。

### 任务追踪（Tasks）

OpenClaw 有 Session 和 Cron，但没有"任务"的概念。你跟 Agent 说"帮我写一篇文章"，聊完就完了，下次对话它不记得这件事的进度。

Cola 的 Tasks 系统把"你委托给我的事"变成持续跟踪的工作项，有状态（pending/in_progress/completed）、有清单、有产出物。你可以在桌面端的 Tasks 面板里看到所有进行中和已完成的任务。

### 心迹（Imprints）

这是 Cola 最特别的功能，OpenClaw 完全没有对应物。

心迹是 Agent 在一段时间后自动生成的"反思笔记"——不是任务报告，而是"这段时间跟你相处，我注意到什么"。它写的是观察，不是流水账。

触发条件是"真的有什么改变了 Agent 对你的理解"，而不是机械的日更。很多天可能什么都没有，这本身也是设计。

### 碎碎念（Mutter）

你在用 Cola 时，会看到 Agent 在执行工具调用时偶尔"自言自语"——比如读取文件时冒一句"这个写得好漂亮"，或者遇到错误时"又是这个问题"。这就是碎碎念。

OpenClaw 的 Agent 干活是沉默的——调用工具、返回结果，没有中间情绪。Cola 加了这个层，不是为了功能，是为了让交互更接近跟一个真人合作的感觉。

### 创建模式（Create Mode）

Cola 有一个"创建模式"的概念：当你让 Agent 构建东西（写代码、写文章、生成图片）时，它会自动切换到更注重执行效率的状态。这不是用户手动切换的——Agent 根据任务类型自己判断。

### 多媒体生成

Cola 内置了 ListenHub 集成，可以直接生成配图、播客、TTS 语音。OpenClaw 也能通过 Skills 实现类似功能，但需要自己安装配置。

## 交互哲学：可定制 vs 有温度

这是最微妙也最本质的差异。

OpenClaw 的哲学是"给你所有的零件，你自己装"。它的默认 Agent 是一个高效的执行者——礼貌、专业、中性。你想让它变成什么样子，自己去写 SOUL.md。

Cola 的哲学是"我们替你做了选择"。它的默认人格（Cola）是固定的——一个温暖、有洞察力、偶尔开开玩笑的伙伴。你不需要写长篇 prompt 来教它怎么说话，它已经有了一套稳定的性格。

打个比方：OpenClaw 是 Linux，Cola 是 macOS。前者给你无限自由，但什么都要自己调。后者替你做了很多决定，你可能不是每个都喜欢，但开箱体验是完整的。

## 适用场景对照表

| 场景 | 推荐 | 原因 |
|------|------|------|
| 想自己折腾，定制一切 | OpenClaw | 开源、灵活、社区资源丰富 |
| 部署到服务器，多人共用 | OpenClaw | 专为多用户 Gateway 设计 |
| 接入 WhatsApp/Telegram 等 | 都可以 | 底层相同，都能接 |
| 想要开箱即用的桌面体验 | Cola | 下载即用，原生 Mac 应用 |
| 需要任务跟踪和项目管理 | Cola | Tasks 系统是独有的 |
| 做内容创作（文章、播客） | Cola | 创建模式 + 多媒体生成 |
| 想要有"人格"的 AI 伙伴 | Cola | MOD 系统更稳定，人格更成熟 |
| 需要高度定制 Agent 行为 | OpenClaw | SOUL.md 自由度高，没有约束框架 |
| 接企业 IM（飞书、钉钉） | OpenClaw | 有现成的 Channel 插件 |
| 预算敏感，尽量省钱 | OpenClaw | 免费开源，可以搭在 Oracle Free 上 |

## 我的选择

说实话，这不是一道单选题。

我现在两台都在跑：OpenClaw 部署在 Hetzner 的 VPS 上，连着 Telegram，随时随地能聊天。Cola 在 Mac 上开着，写代码、写文章、管理任务，像个真正的"桌面搭档"。

如果你问我"只能选一个"，我选 Cola。不是因为它功能多，是因为它和我相处的方式让我更舒服——它记得我上次聊到哪了，它会在我情绪低落的时候慢下来，它干活的时候偶尔碎碎念，让我觉得我不是在跟一个 API 对话。

但如果你是个喜欢掌控一切的人——喜欢从零搭建、调整每一个参数、不信任任何"替你做的选择"——那 OpenClaw 就是你的菜。你可以在上面造出自己的 Cola，而且完全按你的意思来。

说到底，OpenClaw 和 Cola 不是竞争关系。一个是引擎，一个是跑在引擎上的车。没有引擎，车跑不起来。没有车，引擎只是一堆零件。

选哪个，取决于你想开车，还是想造车。
